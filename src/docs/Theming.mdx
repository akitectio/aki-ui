# Theming in @akitectio/aki-ui

@akitectio/aki-ui is built with customization in mind, utilizing Tailwind CSS as its styling foundation. This documentation explains how to customize the appearance of your components to match your brand and design requirements.

## Basic Customization Options

### Using the className prop

All @akitectio/aki-ui components accept a `className` prop that allows you to apply custom Tailwind classes:

```jsx
<Button 
  variant="primary"
  className="bg-indigo-600 hover:bg-indigo-700 font-semibold"
>
  Custom Styled Button
</Button>
```

### Using the style prop

For inline styling needs, you can use the standard React `style` prop:

```jsx
<Card style={{ boxShadow: '0 8px 16px rgba(0, 0, 0, 0.08)' }}>
  <Card.Body>Card with custom shadow</Card.Body>
</Card>
```

```js

          100: '#e0f2fe',
          200: '#bae6fd',
          300: '#7dd3fc',

        'md': '0.375rem',
        'lg': '0.5rem',

    },

  // Enable dark mode
  darkMode: 'class',
  plugins: [],
}
```

## Component-Specific Theming

@akitectio/aki-ui components offer specific props for common styling variations:

### Button Component

```jsx
// Different variants
<Button variant="primary">Primary Button</Button>
<Button variant="secondary">Secondary Button</Button>
<Button variant="outline-primary">Outlined Primary</Button>
<Button variant="ghost">Ghost Button</Button>

// Different sizes
<Button size="sm">Small Button</Button>
<Button size="md">Medium Button</Button>
<Button size="lg">Large Button</Button>
```

### Card Component

```jsx
// Basic card
<Card>
  <Card.Header>Standard Card</Card.Header>
  <Card.Body>Card content</Card.Body>
  <Card.Footer>Card footer</Card.Footer>
</Card>

// Elevated card
<Card elevated>
  <Card.Body>Elevated Card</Card.Body>
</Card>

// Card with custom border
<Card className="border-primary-500 border-2">
  <Card.Body>Custom Border Card</Card.Body>
</Card>
```

### FormControl Component

```jsx
// Different sizes
<FormControl size="sm" label="Small Input" />
<FormControl size="md" label="Medium Input" />
<FormControl size="lg" label="Large Input" />

// With validation states
<FormControl isInvalid errorMessage="This field is required" />
<FormControl isValid successMessage="Looks good!" />
```

## Creating a Theme Provider

For advanced theming needs, you can implement a ThemeProvider to maintain consistent styling across your application:

```jsx
import React, { createContext, useContext } from 'react';

// Create theme context with defaults
const ThemeContext = createContext({
  primaryColor: 'blue-600',
  secondaryColor: 'gray-500',
  accentColor: 'amber-500',
  radius: 'md',
  fontFamily: 'sans',
});

// Theme provider component
export const ThemeProvider = ({ theme, children }) => (
  <ThemeContext.Provider value={{ ...ThemeContext._currentValue, ...theme }}>
    {children}
  </ThemeContext.Provider>
);

// Custom hook for accessing theme values
export const useTheme = () => useContext(ThemeContext);
```

### Using the Theme Provider

```jsx
import { ThemeProvider } from './ThemeContext';
import { Button, Card } from '@akitectio/aki-ui';

function App() {
  const customTheme = {
    primaryColor: 'emerald-600',
    secondaryColor: 'slate-700',
    radius: 'xl',
  };

  return (
    <ThemeProvider theme={customTheme}>
      <div className="p-4">
        <Card>
          <Card.Header>Themed Card</Card.Header>
          <Card.Body>
            <p className="mb-4">This card uses the custom theme</p>
            <Button variant="primary">Themed Button</Button>
          </Card.Body>
        </Card>
      </div>
    </ThemeProvider>
  );
}
```

## Integration with Color Modes

@akitectio/aki-ui supports both light and dark modes. Combine the ThemeProvider with the ColorModeProvider for complete theming control:

```jsx
function AppWithTheming() {
  return (
    <ColorModeProvider>
      <ThemeProvider theme={customTheme}>
        <YourApp />
      </ThemeProvider>
    </ColorModeProvider>
  );
}
```

See the [Color Modes](./ColorModes) documentation for more details on implementing light and dark mode support.

## Best Practices

1. **Consistency**: Define your design tokens in the Tailwind config and reuse them throughout your application.
2. **Accessibility**: Ensure sufficient color contrast between text and backgrounds.
3. **Responsiveness**: Use Tailwind's responsive prefixes to adapt your UI to different screen sizes.
4. **Component Extension**: Create wrapper components that extend @akitectio/aki-ui components with your custom styling when needed.

By following these guidelines, you can create a cohesive and branded experience while leveraging the flexibility and power of @akitectio/aki-ui components.
